# Print session config

YAML files here describe a **single print run** — not subject anatomy (that lives in the bundle).

## Physical landmarks (`pm`)

`physical_landmarks_mm` in `subjects/*.yaml` are **measured at print time** on the real subject. They are never generated by layout_design.

### How to measure

1. Mark three calibration points on the subject's head (same locations as digital picks `landmark_central`, `landmark_left`, `landmark_back`).
2. Mount the **end-effector probe** on the print head (same tool geometry used for printing).
3. Jog the machine so the probe tip touches each marked point in order:
   - Point 1 → origin `[0, 0, 0]` in the print session frame
   - Points 2 and 3 → machine XYZ readout relative to that frame
4. Record the three XYZ tuples into `physical_landmarks_mm` in your session YAML.

The bundle supplies digital scan landmarks (`ps`); `pm` registers scan coordinates to the physical print bed.

## Files

| File | Purpose |
|------|---------|
| `machine_default.yaml` | Machine geometry, speeds, jet timing |
| `subjects/example.yaml` | Template print session |
| `subjects/synthetic.yaml` | Contract tests only |

## Usage

```bash
python -m app convert-gcode \
  --bundle data/output/bundles/subject_{id} \
  --config config/postprocessor/subjects/my_session.yaml
```
